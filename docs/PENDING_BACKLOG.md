## 2. 배치고사 + 랭크 시스템 🔴⭐
**24항목 #19, #20, #21, #22 + 배치고사 기획서 2개 종합**

### 2.1 신병 배치고사 (Initial Calibration) 🔴
- 가입 직후 튜토리얼 없이 60초 극한 테스트
- n+2 알고리즘으로 실시간 난이도 상승
- 3회 연속 오답 또는 반응속도 임계 초과 시 종료
- 결과로 초기 티어 배정

### 2.2 정규 배치고사 (Audition) 🔴
**자격 검정 (Qualification Phase)** — 응시 자격 자체가 리텐션 장치:
- 최근 3~5일 연속 연습 (Streak)
- 현재 레벨에서 음표 500개 이상 판독
- 최근 평균 정확도 90% 이상
- **Pro 구독자 혜택**: 자격 완화 (3일 → 1일)

**평가 산식**:
```
Score = (정답률 × 0.4) + (평균 반응속도 ms × 0.6)
```

### 2.3 티어 시스템 🔴
| 티어 | 세부 등급 | 시각 컨셉 |
|---|---|---|
| **Apprentice (수습)** | 3, 2, 1 | 미니멀 나무 업라이트 피아노 — 기초 음표 |
| **Performer (연주자)** | 3, 2, 1 | 블랙 유광 피아노 + 은빛 — 샵/플랫, 기본 화음 |
| **Virtuoso (거장)** | 3, 2, 1 | 골드 액센트 그랜드 — 복합 텐션, 빠른 템포 |
| **Maestro (마에스트로)** | 단일 (최상위 1%) | 크리스탈/오로라 그랜드 — 챌린지 굿즈 |

각 티어별 디자인 분리 — 레벨 오를수록 시각적 우월감 (24항목 #19)

### 2.4 랭크 감쇠 + 유지 🔴
- 주 3회 미만 연습 → 등급 포인트 하락
- 매월 1회 등급 유지 테스트 통과 필수
- **Pro 혜택**: 강등 보호권 + 스트릭 프리즈

### 2.5 배치고사 진행도 가시화 🟡
**24항목 #20**
- 화면에 "배치고사 응시 자격까지 X% 진행" 표시
- 일별 연속 연습 카운터, 누적 음표 수, 평균 정확도 대시보드
- 동기 부여 + 결제 전환 트리거

### 2.6 레벨 부여 기준 결정 🔴
**24항목 #22**
- 현재 21단계 (Lv 1~7 × Sub 1~3) 시스템과 티어 시스템의 매핑 결정 필요
- 예: Apprentice 3 = Lv 1-1 ~ Lv 1-3? 또는 별도 차원?
- **CTO 의견**: 레벨(콘텐츠 진도) ≠ 티어(실력 평가) 두 차원으로 분리하는 게 자연스러움

### 2.7 DB 스키마 추가 🔴
```sql
-- profiles 테이블에 추가
ALTER TABLE profiles ADD COLUMN current_tier text;
ALTER TABLE profiles ADD COLUMN tier_division integer;
ALTER TABLE profiles ADD COLUMN tier_points integer DEFAULT 0;
ALTER TABLE profiles ADD COLUMN last_test_date timestamp;
ALTER TABLE profiles ADD COLUMN streak_count integer DEFAULT 0;

-- 등급 이력 테이블
CREATE TABLE tier_history (
  id uuid PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id uuid REFERENCES profiles(id),
  from_tier text,
  to_tier text,
  reason text,  -- 'audition', 'decay', 'maintenance', 'admin'
  created_at timestamp DEFAULT now()
);
```

## 7. 성능 · 정밀도 ⭐⭐⭐
**24항목 #13, #23 + 성능 최적화 가이드 + 스트레스 테스트 계획서**

### 7.1 반응 속도 측정 정밀도 🔴
**기획서 명시: 0.00000001초 단위 정밀도**
- `Date.now()` → `performance.now()` 전환 (마이크로초 단위)
- React state → useRef 사용 (리렌더링 회피)
- 입력 이벤트 핸들러 최상단에서 타임스탬프 먼저 기록 (Record First, Process Later)

### 7.2 부동 소수점 오차 방지 🔴
- 시간 비교 시 Big.js 사용 (0.0001초 정밀도 보장)
- 알고리즘 섀도잉: DB raw 데이터를 별도 검증 수식으로 재계산해 1:1 대조

### 7.3 렌더링 최적화 🔴
- `requestAnimationFrame` 활용
- 음표 렌더링: DOM → Canvas 직접 제어 (Virtual DOM 오버헤드 우회)
- 무거운 알고리즘 → Web Worker 분리

### 7.4 네트워크 + 에셋 최적화 🟡
- Edge Functions 도입 (Supabase Edge)
- PWA 구현 (Service Worker로 핵심 에셋 로컬 캐싱)
- 오디오: 고압축 저지연 포맷
- 이미지: WebP/AVIF
- 폰트: Woff2
- 중요 리소스: `<link rel="preload">`

### 7.5 스트레스 테스트 (10만 명 규모) 🟡
**스트레스 테스트 계획서**:
- k6 (Grafana Labs)로 동시접속 부하 테스트
- 0.001초 단위 지터 포함 가상 유저 로그 생성
- Supabase API 가용성 + 락 경합 확인
- 멱등성(Idempotency) 확보 — 네트워크 재전송으로 인한 중복 INSERT 방어

### 7.6 피아노 소리 조정 🔴
- 24항목 #13 — 반응속도와 함께 명시
- 현재 Tone.js 사용 — 음질, 발음 시작 지연 검토 필요
